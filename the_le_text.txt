bất kỳ lệnh build/install nào. Vì v ậy: 
• Tuyệt đối KHÔNG  nộp source code chưa build (không n ộp thư m ục chứa package.json c ần

• Nếu dùng React/Vite/framework có bư ớc build, đ ội phải tự build ("npm run build") trư ớc khi 
nộp, và n ộp đúng thư m ục kết quả build (thư ờng là /dist ho ặc /build) đã đ ổi tên các file vào 
đúng v ị trí theo quy đ ịnh ở mục 2. 
• Nếu dùng Stitch/Bolt.new/Lovable/v0 và xu ất ra HTML/CSS/JS thu ần, có th ể nộp trực tiếp 
các file đó, không c ần thêm bư ớc build.  
2. Cấu trúc thư m ục trong file .zip  
• File index.html PH ẢI nằm ngay t ại thư m ục gốc của file .zip (gi ải nén ra là th ấy 
index.html ngay, khôn g nằm trong thư m ục con).  
• Mọi file CSS, JavaScript, hình ảnh, font... ph ải nằm trong các thư m ục con cùng c ấp với 
index.html (ví d ụ /css, /js, /assets, /images), và đư ợc tham chi ếu bằng đư ờng dẫn tương 
đối (relative path), không dùng đư ờng dẫn tuy ệt đối của máy cá nhân.  
• Zip ch ỉ chứa file ph ục vụ hiển thị trang web (HTML, CSS, JS, hình ảnh, font). Không n ộp 
file project th ừa (node_modules, .git, file c ấu hình IDE...).  
3. Định d ạng & kích thư ớc file

4. Chatbot AI / tích h ợp API

• Đội tự chịu trách nhi ệm đảm bảo API key còn hi ệu lực và còn quota t ại thời điểm thuy ết

• Sản phẩm ph ải kiểm tra đư ợc khi BTC m ở qua URL do BTC c ấp (xem M ục V.4) – nếu

BTC khuy ến khích m ạnh m ẽ mỗi đội tự kiểm tra bài làm b ằng đúng cách BTC s ẽ mở, trước khi

• Đội có trách nhi ệm tự kiểm tra URL đư ợc cấp để chắc chắn bài đã n ộp đúng và ch ạy được. 
BTC không ch ịu trách nhi ệm với các bài n ộp lỗi/thiếu mà đ ội không t ự kiểm tra trư ớc giờ

mỗi đội có m ột URL c ố định riêng ngay t ừ lúc n ộp bài. Lúc lên thuy ết trình:  
• Đội tự dùng laptop c ủa mình, m ở URL do BTC đã c ấp (đội đã bi ết và đã t ự kiểm tra link này 
từ trước) để demo tr ực tiếp cho h ội đồng – BTC không c ần chu ẩn bị máy riêng cho t ừng 
đội, giúp ti ết kiệm thời gian chuy ển đội. 
• Đội mở sẵn VS Cod e (ho ặc editor đang ch ứa code c ủa đội) trên máy, đ ể sẵn sàng khi giám 
khảo yêu c ầu xem code.  
• Trong 5 phút trình bày, đ ội KHÔNG  chủ động trình bày v ề code – chỉ mở và trả lời về code 
khi giám kh ảo trực tiếp yêu c ầu trong ph ần hỏi đáp.  
• Vì lý do trên, vi ệc tuân th ủ đúng c ấu trúc thư m ục và yêu c ầu kỹ thuật ở Mục IV là b ắt buộc 
– bài n ộp sai c ấu trúc có th ể không hi ển thị được khi BTC gi ải nén & serve, ảnh hư ởng trực 
tiếp đến điểm thi c ủa đội.
